Классы

Planet

Абстракный класс планеты. Имеет следующие поля:

  • name : string -- название планеты;
  • coordinates : Coords -- координаты планеты на планетарной карте;
  • steel : float -- коэффициент, на который домножается количество добываемой на планете стали;
  • food : float -- коэффициент, на который домножается количество добываемой на планете еды;
  • money : float -- коэффициент, на который домножается количество добываемых на планете денег;
  • steelHas : int -- количество стали, которое имеет государство планеты в наличии;
  • foodHas : int -- количество еды, которое имеет государство планеты в наличии
  • moneyHas : int -- количество денег, которое имеет государство планеты в наличии
  • description : string -- текстоое описание планеты, которое видит игрок;
  • builds : [Build] -- массив имеющихся на планете построек (см. соответствующий класс Build);
  • image : string --  имя изображения, соответствующего планете;
  • master : int -- номер игрока, которому принадлежит планета; нейтральные планеты имеют хозяина -1;

Важно: т.к. класс является абстрактным, то не рекомендуется создавать объекты его типа. Поля коэффициентов будут иметь значение None. Предпочтительнее использовать классы конкретизированных планет, которые описаны ниже:

Важно #2: надо импортировать Types из модуля planetClass

Важно №3: названия картинок содержатся в словаре img в начале модуля (для того, чтобы к ним не было доступа извне). Для того, чтобы добавить планете название картинки, просто занесите его в словарь: длина обновляется автоматически.

  • Types.lave -- лавовая планета (см. таблицу планет в документации);
  • Types.water -- водяная планета (см. таблицу планет в документации);
  • Types.ice -- ледяная планета (см. таблицу планет в документации);
  • Types.desert -- пустынная планета (см. таблицу планет в документации);
  • Types.earth -- планета Земного типа (см. таблицу планет в документации);
  • Types.air -- газовая планета (см. таблицу планет в документации);
  • Types.rock -- скалистая планета (см. таблицу планет в документации);

Важно №4: в модуле planetClass также присутствует словарь typeToClass, который восстанавливает обратно тип планеты по его значению из Types:

  • "0" : Lave;
  • "1" : Air;
  • "2" : Ice;
  •  3" : Earth;
  • "4" : Desert;
  • "5" : Rock;
  • "6" : Water;

Все они наследуют класс Planet, переопределяя конструктор и некоторые методы из описанных ниже (см. ограничения на кол-во построек для планет).

Методы класса Planet:

  • __init__(name, coordinates) -- конструктор, на вход получает значения одноимённых полей;
  • getSteel() : int -- геттер, возвращает значение поля steel;
  • getSteelHas() : int -- геттер, возвращает значение поля steelHas;
  • getFoodHas() : int -- геттер, возвращает значение поля foodHas;
  • getMoneyHas() : int -- геттер, возвращает значение поля moneyHas;
  • getFood() : int -- геттер, возвращает значение поля food;
  • getMoney() : int -- геттер, возвращает значение поля money;
  • getFood() : int -- геттер, возвращает значение поля food;
  • setSteelHas(value : int)  -- сеттер, устанавливает значение поля steelHas в value;
  • setFoodHas(value : int)  -- сеттер, устанавливает значение поля foodHas в value;
  • setMoneyHas(value : int) -- сеттер, устанавливает значение поля moneyHas в value;
  • setImage(value : int) -- сеттер, устанавливает устанавливает поля image в value;
  • addMoney(value : int) -- прибавляет value к значению поля moneyHas;
  • addFood(value : int) -- прибавляет value к значению поля foodHas;
  • addSteel(value : int) -- прибавляет value к значению поля steelHas;
  • getImage() : string -- геттер, возвращает значение поля image;
  • getPort() : [Port] -- геттер, возвращает все Port на планете ;
  • next_turn() -- переводит игру на следующий ход;
  • getDefence() : [Defence] -- геттер, возвращает все Defence на планете ;
  • getFun() : [Fun] -- геттер, возвращает все Fun на планете ;
  • getFarm() : [Farm] -- геттер, возвращает все Farm на планете ;
  • getMain() : [Main] -- геттер, возвращает все Main на планете ;
  • getCoordinates() : Coords -- геттер, возвращает Coords планеты ;
  • getBuildsLen() : int -- типа геттер, возвращает суммарное количество строений на планете;
  • addPort(port) : bool -- пытается построить порт, возвращает True (удалось) / False (нельзя); в случае успеха добавляет port в поле builds;
  • addDefence(defence) : bool -- пытается построить планетарный гарнизон, возвращает True (удалось) / False (нельзя); в случае успеха добавляет defence в поле builds;
  • addFun(fun) : bool -- пытается построить развлекательное сооружение, возвращает True / False (нельзя); в случае успеха добавляет fun в поле builds;
  • addFarm(farm) : bool -- пытается построить ферму, возвращает True (удалось) / False (нельзя); в случае успеха добавляет farm в поле builds;
  • addMain(main) : bool -- пытается построить шахту, возвращает True (удалось) / False (нельзя); в случае успеха добавляет main в поле builds;
  • cache() : string -- переводит всю информацию о планете в текстовую форму. Всё выводится без пояснений, в столбик (метод необходим для реализации сохранений в будущем). 
    • Порядок вывода следующий:
      • type;
      • name;
      • coordinates;
      • steelHas;
      • foodHas;
      • moneyHas;
      • image;
      • master;
    • Замечание: передаются только уникальные параметры; остальное можно восстановить, зная значение поля type;
  • static typeToStr(planetType : int) : string -- возвращает название типа планеты (пока только на русском языке, однако);

ФУНКЦИЯ: readPlanet(f : file) : Planet -- считывает с файла f планету (в том же порядке, что и кеширует cache) и возвращает полученную планету.

Важно: также перегружен перевод планеты в строку (для удобства работы с выводом в файл). Всё выводится с комментариями, всё читаемо (для логгирования / консольной графики -- самое то).

Star

Класс, описывающий звезду. Имеет следующие поля:

  • name : string -- имя звезды;
  • planets : [Planet] -- планеты, принадлежащих звезде;
  • neighbours : (int, int) -- смежные с оной звёзды: первое число -- длина пути, второе -- номер звезды, к которой путь ведёт, в графе; для читаемости кода в классе звезды Star есть методы для работы с оным кортежем: static getNeighbour(way : (int, int)) и static getWayLen(way : (int, int));
  • sizex : int -- размеры планетарной карты по оси х; 
  • sizey : int -- размеры планетарной карты по оси х;

Методы класса следующие:

  • конструктор( name : string, neighbours : [ (int, int) ] ) -- создаёт новую звезду; на вход принимает массив звёзд-соседей neighbours и имя name; 
  • static getNeighbour( way : (int, int) ) : int -- возвращает номер звезды-соседа, к которой ведёт путь way;
  • static getWayLen( way : (int, int) ) : int -- возвращает длину пути way;
  • setSize(sizex : int, sizey : int) -- сеттер; устанавливает полям с соответствующими именами передаваемые значения;
  • getName() : string -- геттер; возвращает значение поля name;
  • getNeighbours() : [ (int, int) ] -- геттер; возвращает значение поля neighbours;
  • setPlanets( planets : [Planet] ) -- сеттер; устанавливает полям с соответствующими именами передаваемые значения;
  • planetsNumber() : int -- геттер; возвращает число планет, принадлежащих звезде;
  • str(other : [Star]) : string -- переводит всю информацию о звезде в текстовую форму. На вход принимает массив звёзд stars из gameMap, которому принадлежит звезда; всё выводится с пояснениями, где какое значение и т.п. (т.е, является читаемым для человека. Возможно применение для логгирования или разработки консольной графики);
  • cache() : string -- переводит всю информацию о звезде в текстовую форму. Всё выводится без пояснений, в столбик (метод необходим для реализации сохранений в будущем); 
    • Порядок вывода следующий:
      • name;
      • len(neighbours);
      • [neighbours]:
        • 0;
        • 1;
      • len(planets);
      • [planets];
  • static readStar(f : file) : Star-- считывает с файла f звезду (в том же порядке, что и кеширует cache) и возвращает полученную звезду;

 

gameMap

Поля:

  • stars : [Star] -- звёзды на карте;
  • ships : [Ship] -- корабли на карте (класс пока не реализован, увы и ах);
  • sizex : int -- размер планетарной карты по оси x (скоро может быть удалён, если будет принято решение о произвольности размеров планетарной карты для каждой звезды);
  • sizey : int -- размер планетарной карты по оси y (скоро может быть удалён, если будет принято решение о произвольности размеров планетарной карты для каждой звезды); 

Методы:

  • Конструктор на вход не принимает ничего (это связано с особенностями генерации карты). Все поля по умолчанию равны None.
  • setSize(sizex : int, sizey : int) -- сеттер; устанавливает полям с соответствующими именами передаваемые значения; кроме того он устанавливает всем членам массива stars эти же значения (поля называются sizex и sizey);
  • setStars( stars : [Star] ) -- сеттер; устанавливает полям с соответствующими именами передаваемые значения; кроме того он устанавливает всем членам массива stars эти же значения полей sizex и sizey в значения одноимённых полей текущего объекта (ВАЖНО: если их значение не равно None);
  • setShips( ships : [Ship] ) -- сеттер; устанавливает полям с соответствующими именами передаваемые значения (класс Ship пока не реализован, увы и ах);  
  • cache() : string -- переводит всю информацию о карте в текстовую форму. Всё выводится без пояснений, в столбик (метод необходим для реализации сохранений в будущем); 
    • Вывод реализован в следующем порядке:
      • sizex;
      • sizey;
      • len(stars);
      • [stars];
  • static readMap(name : str) : gameMap -- открывает файл "name", считывает (в том же порядке, что и кеширует cache) и возвращает полученную карту;

Важно: также перегружен перевод карты в строку (для удобства работы с выводом в файл). Всё выводится с комментариями, всё читаемо (для логгирования / консольной графики -- самое то).
